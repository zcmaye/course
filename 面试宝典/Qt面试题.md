# C++ Qt 开发岗面试题（含答案）

我整理了**高频基础、C++ 核心、Qt 核心、项目实战、手写代码**五类面试题，覆盖校招 / 社招初面、技术面，答案简洁精准，直接背即可。

## 一、基础必问（10 题）

### 1. 什么是 Qt？它的核心特点是什么？

**答案**：Qt 是跨平台 C++ 图形用户界面（GUI）开发框架，支持 Windows/Linux/macOS/ 嵌入式等多平台，一次编写多平台编译运行。

核心特点：**信号与槽机制**、跨平台、丰富的 UI 组件、支持多线程 / 网络 / 数据库、MVC 架构、元对象系统（MOC）。

### 2. Qt 的元对象系统（MOC）是什么？

**答案**：Qt 基于 C++ 扩展的元对象系统，由 MOC 预处理器生成代码，实现**信号与槽、属性系统、动态类型转换、反射**。

必须满足：类继承 QObject、添加`Q_OBJECT`宏、使用 Qt 关键字（signals/slots）。

### 3. 信号与槽是什么？和回调函数的区别？

**答案**：

- 信号：对象状态改变时发出的通知；
- 槽：接收信号的处理函数；
- 连接：`connect(发送者, SIGNAL(信号), 接收者, SLOT(槽))` / 新语法`connect(sender, &Sender::signal, receiver, &Receiver::slot)`。

区别：信号与槽**类型安全、解耦**，回调函数是强耦合，无类型检查。

### 4. Qt 中`connect`的第五个参数（连接类型）有哪些？

**答案**：

1. `Qt::AutoConnection`（默认）：自动判断，同线程用直连，跨线程用队列；
2. `Qt::DirectConnection`：直连，信号发出立即执行槽函数（同线程）；
3. `Qt::QueuedConnection`：队列连接，接收者线程事件循环执行（跨线程必用）；
4. `Qt::BlockingQueuedConnection`：阻塞队列，发送线程等待槽执行完毕（跨线程慎用）。

### 5. `QObject`的父子机制作用？

**答案**：

1. 自动内存管理：父对象销毁时，**子对象自动销毁**，避免内存泄漏；
2. 层级管理：UI 控件父子关系对应界面层级；
3. 事件传递：事件从子对象向父对象传递。

### 6. Qt 内存泄漏如何避免？

**答案**：

1. 优先使用**父子对象机制**；
2. 局部控件用`std::unique_ptr`/`QScopedPointer`智能指针；
3. 手动`new`的对象必须配对`delete`；
4. 跨线程对象必须在所属线程销毁。

### 7. 模态窗口和非模态窗口的区别？

**答案**：

- 模态：`exec()`，阻塞父窗口，必须关闭才能操作其他窗口；
- 非模态：`show()`，不阻塞，可同时操作多个窗口。

### 8. Qt 布局管理器有哪些？

**答案**：

1. `QVBoxLayout`：垂直布局；
2. `QHBoxLayout`：水平布局；
3. `QGridLayout`：网格布局；
4. `QFormLayout`：表单布局（标签 + 输入框）。

### 9. 什么是 Qt 的事件循环？

**答案**：`QApplication::exec()`启动的无限循环，负责接收**鼠标 / 键盘 / 定时器 / 信号**等事件，分发给对应对象处理，是 Qt 界面响应的核心。

### 10. `QString`和`std::string`的区别？

**答案**：

- `QString`：Unicode 编码，支持多语言，Qt 原生类型，自带丰富字符串处理函数；
- `std::string`：ASCII/UTF-8，标准库类型，无 UI 相关接口；
- 转换：`QString::toStdString()` / `QString::fromStdString()`。

## 二、C++ 核心面试题（Qt 岗必考）

### 1. 什么是虚函数？纯虚函数呢？

> 考察对 C++ 多态核心机制的理解。

- **虚函数**：在基类中使用 `virtual` 关键字修饰的成员函数。它支持**动态绑定**，即允许通过基类指针或引用调用派生类中重写（override）的函数，从而实现运行时多态。
- **纯虚函数**：在虚函数声明末尾加上 `= 0`。包含纯虚函数的类称为**抽象类**，它不能实例化对象。纯虚函数要求所有派生类都必须提供自己的实现，通常用于定义接口规范

### 2. 虚析构函数的作用？

**答案**：基类指针指向子类对象时，**delete 基类指针能调用子类析构函数**，防止内存泄漏。Qt 中`QObject`析构函数就是虚函数。

### 3. 请解释 `std::move` 和移动语义。

> 考察对 C++11 性能优化的理解。

- **移动语义**：允许将资源（如堆内存）从一个对象“移动”到另一个对象，而不是进行昂贵的深拷贝。这极大地提高了处理临时对象或大型容器的效率。
- **`std::move`**：它本身并不移动任何东西，仅仅是一个**类型转换**工具，无条件地将左值（lvalue）转换为右值引用（rvalue reference），从而“提示”编译器可以对该对象进行移动操作。

### 4. 指针和引用的区别？

**答案**：

1. 指针可空、可修改指向；引用必须初始化、不可更改指向；
2. 指针有多级指针，引用没有；
3. 指针需要解引用`*`，引用直接使用。

### 5. 重载、重写、隐藏的区别？

**答案**：

- 重载：同一作用域，函数名相同、参数不同；
- 重写：子类重写基类**虚函数**，函数签名完全一致；
- 隐藏：子类函数与基类同名，屏蔽基类函数。

### 6. C++11 中引入了哪几种智能指针？它们有什么区别？

> 考察对现代 C++ 内存管理的掌握程度。

- **`std::unique_ptr`**：**独占所有权**的智能指针。它不允许拷贝，只能通过 `std::move` 转移所有权。当你需要一个指针拥有某个对象且不被共享时使用，开销极小。
- **`std::shared_ptr`**：**共享所有权**的智能指针。它使用**引用计数**来管理资源，当最后一个 `shared_ptr` 被销毁时，对象被释放。适用于多个对象需要同时访问同一块资源的场景。
- **`std::weak_ptr`**：**弱引用**指针，配合 `shared_ptr` 使用。它不增加引用计数，主要用于解决 `shared_ptr` 相互引用导致的**循环引用**问题，从而避免内存泄漏。

### 7. 解释一下 RAII 是什么？它在 C++ 中有什么作用？

> 这是 C++ 独特的资源管理思想，几乎是必考题。

- **定义**：RAII 全称 Resource Acquisition Is Initialization（资源获取即初始化）。它是一种 C++ 编程惯用法，将资源的生命周期与对象的生命周期绑定。
- **原理**：在**构造函数**中申请资源（如分配内存、打开文件、加锁），在**析构函数**中释放资源。这样，当对象离开作用域时，其析构函数会被自动调用，从而保证资源被正确释放。
- **作用**：主要用于**防止资源泄漏**（特别是内存泄漏），并保证代码的异常安全性。智能指针（如 `std::unique_ptr`）就是 RAII 的典型应用。

## 三、Qt 核心进阶题（技术面重点）

### 1. 请详细解释 Qt 的信号与槽（Signals & Slots）机制。

> Qt 的灵魂，必须深入理解。

- **定义**：信号与槽是一种对象间通信的机制。当一个对象的状态发生改变时，它会**发射（emit）** 一个信号，而另一个对象可以有一个**槽函数**来响应这个信号。一个信号可以连接多个槽，多个信号也可以连接同一个槽。
- **优点**：
  - **松耦合**：发送信号的对象不需要知道哪些对象（接收者）的哪些函数（槽）会接收该信号。这种机制极大地降低了对象之间的耦合度。
  - **类型安全**：信号的签名必须与接收槽的签名相匹配（槽的参数可以少于信号，但不能多于信号）。
- **本质**：信号和槽函数最终由**元对象编译器（MOC，Meta-Object Compiler）** 处理成普通的 C++ 函数和回调机制。

### 2. 信号槽的连接方式有哪些？有什么区别？

> 考察对多线程和连接类型细节的掌握。

- **`Qt::AutoConnection`**（默认）：**自动判断**。如果发送者和接收者在同一线程，自动使用 `DirectConnection`；如果在不同线程，则使用 `QueuedConnection`。
- **`Qt::DirectConnection`**：**同步调用**。信号发射后，槽函数会立即在**发送者所在的线程**中执行。这本质上是函数直接调用。
- **`Qt::QueuedConnection`**：**异步调用**。信号发射后，其参数会被拷贝并封装成一个事件，放入**接收者线程**的事件队列中。接收者线程的事件循环会在稍后处理这个事件并调用槽函数。这种方式是线程安全的。
- **`Qt::BlockingQueuedConnection`**：**同步阻塞**。与 `QueuedConnection` 类似，但发送者线程会**阻塞**，直到接收者线程的槽函数执行完毕。**注意**：不能在发送者与接收者在同一线程中使用，否则会死锁。

### 3. Qt 中如何进行多线程编程？如何实现线程间通信？

> 考察实际并发编程能力。

- **实现方式**：
  - **方法一（推荐）**：创建一个 `QObject` 的子类（工作者对象），实现耗时逻辑。然后创建 `QThread` 对象，使用 `moveToThread()` 方法将工作者对象移动到新线程中。这种方法更安全，因为它将线程逻辑与业务逻辑分离。
  - **方法二**：继承 `QThread` 并重写其 `run()` 函数。`run()` 中的代码将在新线程中执行。这种方法更简单直接，但需要小心处理对象生命周期。
- **线程间通信**：最安全、最推荐的方式是使用 **信号与槽**，并选择 `Qt::QueuedConnection` 连接方式。这样，在不同线程间传递数据时，Qt 会自动保证槽函数的调用是线程安全的（例如，通过排队和拷贝参数）。

**注意**：UI 控件**只能在主线程创建 / 操作**，子线程禁止操作 UI。

### 4. 什么是 Qt 的元对象系统（Meta-Object System）？它提供了哪些功能？

> 考察对 Qt 框架底层扩展的理解。

- **定义**：元对象系统是 Qt 对标准 C++ 的扩展，它使得 Qt 能够实现信号与槽、运行时类型信息（RTTI）、动态属性等高级功能。
- **三要素**：
  1. **基类 `QObject`**：所有需要使用元对象系统特性的类都必须直接或间接继承自 `QObject`。
  2. **宏 `Q_OBJECT`**：在类私有声明区使用该宏，它会启用元对象特性，如动态属性、信号槽等。
  3. **元对象编译器（MOC，Meta-Object Compiler）**：一个预处理工具，它会读取带有 `Q_OBJECT` 宏的类，并生成相应的 C++ 代码文件（如 `moc_*.cpp`），其中包含了元对象所需的代码。

### 5. 描述一下 Qt 的事件循环（Event Loop）是什么？

> 考察对 GUI 应用程序驱动模型的理解。

- **定义**：事件循环是一个无限循环（`QCoreApplication::exec()`），它不断地从线程的事件队列中取出事件，并将它们分发给相应的目标对象进行处理。
- **工作原理**：
  1. 当用户点击鼠标、按下键盘或系统产生定时器超时等事件时，操作系统会将事件传递给 Qt 应用程序。
  2. Qt 的窗口系统接口会捕获这些事件，并将其转换为 `QEvent` 对象，放入对应线程的事件队列中。
  3. 主线程的事件循环不停地在运行，它会从队列中取出一个事件，并调用 `QObject::event()` 函数将其发送给目标对象。
  4. 目标对象收到事件后，会调用相应的事件处理器（如 `mousePressEvent` 或更通用的 `event()` 函数）进行响应。
- **阻塞**：如果在主线程的事件循环中执行一个耗时操作（如 `sleep` 或大量计算），那么事件循环就无法及时处理队列中的其他事件（如界面重绘、用户输入），导致界面**“卡死”**或**“无响应”**。

### 6. 谈谈你对 MVC / 模型/视图（Model/View）框架的理解。

> 考察对 GUI 架构设计的理解。

- **概念**：模型/视图框架是一种将**数据（Model）** 与**界面展示（View）** 分离的架构。
  - **模型（Model）**：负责与数据源（如数据库、文件、字符串列表）交互，并提供标准接口（如 `QAbstractItemModel`）供视图访问数据。模型不关心数据如何显示。
  - **视图（View）**：负责从模型中获取数据并将其显示给用户（如 `QListView`, `QTableView`）。视图不关心数据从何而来。
  - **委托（Delegate）**：负责处理用户在视图上的编辑操作（如修改单元格内容），并控制项的绘制方式和大小。
- **优点**：这种分离使得同一份数据可以在不同的视图上展示（如表格和树形），或者在不修改数据源的情况下改变数据的显示方式，极大地提高了代码的复用性和可维护性。

## 四、项目实战 / 场景题

### 1. 界面卡顿如何优化？

**答案**：

1. 耗时操作（网络 / 文件 / 大量计算）放到**子线程**，避免阻塞主线程事件循环；
2. 减少 UI 刷新频率，使用`update()`合并重绘请求，避免频繁调用`repaint()`；
3. 复杂列表 / 表格使用`QTableView` / `QListView` + 自定义模型（MVC），避免创建过多真实控件；
4. 图片加载时进行压缩或缩放，避免在 UI 线程中直接加载大图；
5. 使用`QElapsedTimer`或性能分析工具（QElapsedTimer / Valgrind / perf）定位瓶颈。

### 2. 跨线程更新 UI 怎么做？

**答案**：

1. 子线程**不能直接操作 UI**（非线程安全）；
2. 子线程通过**信号槽**发送数据到主线程，连接方式必须为`Qt::QueuedConnection`（或自动）；
3. 主线程的槽函数中执行实际的 UI 更新操作；
4. 也可以使用`QMetaObject::invokeMethod()`并指定`Qt::QueuedConnection`。

### 3. Qt 程序发布流程？

**答案**：

1. 使用 **Release 模式**编译；
2. 使用对应平台的部署工具：
   - Windows：`windeployqt` 自动拷贝依赖的 dll 和插件；
   - Linux：`linuxdeployqt`；
   - macOS：`macdeployqt`；
3. 手动验证依赖完整性（可通过 Dependencies / ldd / otool 检查）；
4. 打包成安装包（Inno Setup / InstallShield / NSIS / CPack）。

### 4. 如何实现软件换肤 / 样式？

**答案**：

1. 使用 **Qt Style Sheet（QSS）**，语法类似 CSS；
2. 通过`setStyleSheet()`直接设置或加载外部 `.qss` 文件；
3. 支持统一定义控件样式（颜色、字体、边框、背景、图标等）；
4. 切换皮肤时动态加载不同 qss 文件，并调用`qApp->setStyleSheet()`刷新；
5. 可进一步封装主题管理器，支持运行时切换。

## 五、手写代码题（高频）

### 1. 自定义信号与槽

```cpp
// myclass.h
#include <QObject>
class MyClass : public QObject
{
    Q_OBJECT
signals:
    void sendMsg(const QString& msg); // 信号

public slots:
    void recvMsg(const QString& msg); // 槽
};

// myclass.cpp
void MyClass::recvMsg(const QString& msg)
{
    qDebug() << "收到：" << msg;
}

// 调用
MyClass obj1, obj2;
connect(&obj1, &MyClass::sendMsg, &obj2, &MyClass::recvMsg);
emit obj1.sendMsg("Hello Qt");
```

### 2. Qt 多线程（moveToThread 方式）

```cpp
// 工作类
class Work : public QObject
{
    Q_OBJECT
public slots:
    void doWork() {
        // 耗时逻辑
        emit workFinished();
    }
signals:
    void workFinished();
};

// 主线程使用
QThread* thread = new QThread;
Work* work = new Work;
work->moveToThread(thread);
connect(thread, &QThread::started, work, &Work::doWork);
connect(work, &Work::workFinished, thread, &QThread::quit);
thread->start();
```

### 3. Qt 定时器

```cpp
QTimer* timer = new QTimer(this);
connect(timer, &QTimer::timeout, [](){
    qDebug() << "定时触发";
});
timer->start(1000); // 1秒触发一次
```
